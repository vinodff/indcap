# CEO Plan: AI Auto Video Enhancement Engine

Generated by `/plan-ceo-review` on 2026-06-14
Branch: main | Mode: SELECTIVE EXPANSION
Approach: Hybrid (client-side WebGL visual + free WASM audio; cloud audio skipped)

> One-line promise: **upload a raw phone clip → one tap → it looks like a production team edited it.** Free and instant for the visual pass; clean audio in-browser at no cost.

---

## 1. Vision

### 10x check
The win is not "a filters panel." It's that every other feature you already have — captions, auto-camera, SFX, thumbnails — gets better automatically because they now run on a *clean* input instead of the user's worst phone footage. Enhancement sits **upstream of the whole product**. That is the leverage.

### The felt experience
User drops in a dim, noisy, slightly-blurry selfie clip. Within ~3 seconds a quiet readout appears: **"Quality 48/100."** Next to it a Studio Mode button and a before/after slider. They drag the slider — the right half is brighter, color-true, face sharp, audio hiss gone. They tap **✨ Studio Mode**. Score jumps to **91/100**. They export straight to a TikTok-ready file. No timeline, no sliders, no editing knowledge.

### Accepted v1 scope (build this)
1. **EnhancementEngine** — WebGL pre-pass: smart color correction (auto-levels from luminance histogram, white-balance, contrast/saturation), light spatial denoise (bilateral), MediaPipe face mask → selective sharpen + gentle skin smoothing.
2. **Scene classifier** — Gemini vision tags `indoor | outdoor | night | gaming | podcast | educational` → selects a tuned parameter preset (LUT-equivalent).
3. **Studio Mode UI** — auto-analyze on upload, Quality Score readout, before/after slider, one-tap apply.
4. **Quality Score 0–100** — no-reference metric: sharpness (Laplacian variance), exposure (histogram clipping), contrast spread, + audio SNR estimate. Shown before vs after.
5. **AudioEnhancer** — RNNoise WASM on the audio track, wired into the MediaRecorder audio path.
6. **Platform export presets** — one tap sets aspect/resolution/bitrate/safe-zone for YT, Insta, TikTok, FB.

## 3. Architecture (Section 1 finding — DECIDED)

**Decision: WebGL pre-pass into an offscreen canvas.** Not a full renderer rewrite. Smallest correct diff, every existing feature keeps working.

```
                 ┌──────────────────────────────────────────────┐
 upload ──▶ <video>                                              │
                 │                                               │
                 ▼   NEW — services/enhance/EnhancementEngine.ts │
        ┌────────────────────────┐                               │
        │ WebGL/WebGPU shader chain│  ◀── params from:            │
        │  • white-balance/levels  │      • SceneClassifier(Gemini)
        │  • contrast/saturation   │      • Studio Mode toggle    │
        │  • bilateral denoise     │                              │
        │  • face mask sharpen ◀───┼── @mediapipe/tasks-vision    │
        └───────────┬─────────────┘                               │
                    ▼                                              │
            [offscreen GL canvas]                                 │
                    │                                              │
                    ▼  existing captionRenderer.ts:372 — one swap │
   ctx.drawImage(offscreenCanvas, r.sx, r.sy, r.sw, r.sh, ...)   │
                    │  (camera source-rect still applies AFTER)   │
                    ▼                                              │
            [2D caption canvas]  captions · emojis · SFX UNTOUCHED │
                    │                                              │
                    ▼  App.tsx:1134 captureStream → MediaRecorder │
              exported file (enhancement baked in for free) ──────┘
```

Key properties:
- **One insertion point.** Line 372 swaps `video` → `offscreenCanvas`. Camera crop, captions, export untouched.
- **Reversible.** Studio Mode off = skip the pass, `drawImage(video)` as today. Feature-flag-grade rollback with zero migration.
- **Export is free.** MediaRecorder already captures the final 2D canvas, so enhancement bakes into export with no new export code (same pattern as the auto-camera bake at captionRenderer:361).

### Data flow — four paths
```
INPUT(video frame) ─▶ VALIDATE ─▶ GL ENHANCE ─▶ COMPOSITE(line372) ─▶ EXPORT
   │                    │             │                │                 │
   nil  → skip pass,    bad dims →    GL ctx lost →    offscreen not     MediaRecorder
   draw nothing(early   skip pass,    drop to          ready → draw      unsupported mime
   return, no crash)    log once      drawImage(video) raw video         → existing fallback
   empty(0×0) →         (no GL on this frame, log once, never throw inside rAF)
```

## 4. Error & Rescue Map (Section 2)

| Codepath | What can go wrong | Exception/condition | Rescued? | Action | User sees |
|---|---|---|---|---|---|
| `EnhancementEngine.init()` | WebGL2/WebGPU unavailable | `getContext` returns null | Y | Disable engine, set `enhanceSupported=false` | "Enhancement not supported on this browser" badge; raw video still plays |
| `EnhancementEngine.process(frame)` | GL context lost (tab background/GPU reset) | `webglcontextlost` event | Y | Fall back to `drawImage(video)` this frame; attempt restore | Brief unenhanced frames, no crash |
| shader compile | driver rejects shader | `getShaderParameter` false | Y | Log full shader log once, disable that pass | That one pass skipped, others apply |
| `SceneClassifier.classify()` | Gemini 429 / 404 / timeout | network/quota | Y | Fall back to a neutral default preset | Enhancement still applies (generic look) |
| `SceneClassifier.classify()` | Gemini returns non-enum / malformed JSON | parse/validation | Y | Validate against enum allowlist; default on miss | Generic preset, no error |
| `AudioEnhancer` (RNNoise) | WASM fails to load | fetch/instantiate error | Y | Bypass denoise, pass original audio track | Audio unprocessed, video still enhances |
| `AudioEnhancer` | no audio track in file | empty track | Y | Skip audio stage entirely | Visual-only enhancement |
| `QualityScore.compute()` | frame not yet decoded | readPixels empty | Y | Defer to next frame; show "Analyzing…" | Spinner, then score |
| Studio Mode apply | rapid toggle / re-upload mid-analyze | stale async result | Y | Guard with a request token; ignore stale | Latest state always wins |

**No catch-all.** Every stage degrades to "pass the original through" rather than throwing inside the `requestAnimationFrame` loop. A failed enhancement must never break playback or export.

## 6. Performance (Section 7 — the real risk)

The export path is **realtime MediaRecorder** (`canvas.captureStream(fps)`). Enhancement must keep per-frame work under the frame budget (~33ms @30fps) or export drops frames / runs slow.

Mitigations baked into the plan:
- Process at **display resolution**, not source (don't enhance 4K then downscale).
- **Classify scene once** per clip (or per scene cut), cache the preset — never per-frame Gemini calls.
- Shaders are single-pass where possible; denoise kernel size capped.
- Quality score sampled on **keyframes only** (e.g., every 15th frame), not every frame.
- Super-resolution explicitly **deferred** — it's the one stage that blows the frame budget.
- Perf guard: if measured frame time exceeds budget for N frames, auto-drop the heaviest pass (denoise) and surface a one-time "reduced enhancement for performance" note.

## 7. UX (Section 11 — DECIDED: auto-analyze + one-tap)

```
 ┌─ on upload ───────────────────────────────────────────┐
 │ analyze (2–3s, non-blocking)                           │
 │   ▼                                                    │
 │  Quality: 48/100   ▓▓▓▓▓░░░░░                          │
 │  [ ◀ Before | After ▶ ]  live slider over the preview  │
 │  ✨ Apply Studio Mode      → score 91/100              │
 │  Scene detected: Indoor · low light                    │
 │  Export ▾  [ YouTube · Instagram · TikTok · Facebook ] │
 └────────────────────────────────────────────────────────┘
```

State coverage:
| State | Handling |
|---|---|
| LOADING | "Analyzing…" shimmer on the score chip |
| EMPTY (no enhancement possible / unsupported) | score hidden, "Enhancement unavailable" badge, raw playback |
| ERROR (a stage failed) | silent degrade, badge "Some enhancements skipped" |
| SUCCESS | before/after slider + new score |
| PARTIAL (audio failed, visual ok) | visual applies, small "audio unchanged" note |

Edge cases: re-upload mid-analyze (request token guard), toggle spam (debounced), navigate-away (cancel async), very short clip / single frame (score from available frames).

## 9. NOT in scope (deferred, with rationale)
- **AI super-resolution upscale** — perf risk; ship as opt-in "Enhance Resolution" toggle after v1 perf is proven.
- **Cloud voice isolation (ElevenLabs)** — needs a premium tier / billing that doesn't exist yet.
- **AI video director / auto-highlights & clip generation** — large separable subsystem; its own project (overlaps existing camera "moment" detection — reuse that).
- **Excessive-filter detection & removal** — needs bespoke skin-tone/over-saturation heuristics; niche.

## 10. What already exists (reused)
- `@mediapipe/tasks-vision` → face mask (no new dep).
- `@google/genai` + `services/geminiService.ts` → scene classification (no new dep).
- `captionRenderer.ts:372` render chokepoint + camera source-rect → single clean insertion point.
- `App.tsx` MediaRecorder export → enhancement bakes in free.
- Multi-agent "director → params → render hook" pattern from camera/audio subsystems → mirror it.

## 11. Dream-state delta
12-month ideal: upload → fully automatic professional creator video (clean, color-graded, upscaled, captioned, thumbnailed, platform-exported). This v1 delivers the **visual + audio + scene + export** core and the **one-tap + before/after proof** that makes it believable. Super-res and the auto-director are the next two phases on the same insertion point — the architecture is built to receive them without rework.
